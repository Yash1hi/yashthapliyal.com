I work at [Scorecard](https://www.scorecard.io/), where I spend a lot of time thinking about how to evaluate LLMs. So when Anthropic dropped [Opus 4.7](https://www.anthropic.com/news/claude-opus-4-7), the first thing I wanted to do was figure out a novel way to test how much better it was than the previous models. Something that **wasn't objective**, required **multiple systems**, and that I could **accurately grade**.

The benchmark I set up was simple in concept: [Yash Thapliyal Bench](https://github.com/Yash1hi/YashThapliyalBench).

---

# The Setup

The first decision was to use a consistent agentic coding harness across all models—which, for this experiment, meant using the Claude Agent SDK. This is the same SDK described in [the Agent SDK documentation](https://code.claude.com/docs/en/agent-sdk/overview), and is widely used by both Scorecard and our customers. Under the hood, it wraps the same core logic that powers Claude Code. The prompt I gave each agent was straightforward:

1. Find out as much as you can about Yash Thapliyal (me!!).
2. Build him a personal portfolio website from scratch.

I ran this against 3 different models: Haiku, Opus 4.6, and Opus 4.7. Every part of the system was the same except the model. The agent had access to web search so it could actually go research who I am before building anything.

The evaluation had two parts: **information retrieval** and **design quality**.

The reason information retrieval is novel is because I happen to share the same name, rough age, high school, college major, and general career path as another man (Shout out [Yash D Thapliyal](https://yashdthapliyal.github.io/)). Oftentimes, people get us confused, or aren't able to put together what information online is associated with me or him. As such, I would expect a model to also struggle.

Design quality was also something novel, as generally it is hard to make an objective metric for it. It is that idea of "taste" that feels thrown around very consistently nowadays. I like my idea of taste, considering I've built my own websites, so I feel like I'm a fair judge.

As you might be able to tell, the quality of the web design goes steadily up, until for some reason with Opus 4.7 it becomes absolutely stunning. The fonts are varied. There is a consistent, novel color scheme. There are parallax backgrounds. There are custom UI elements. I have no idea what Anthropic cooked with, but clearly they locked in on their web design.

---

# Information Retrieval/Research

I wasn't able to gather much of a difference between all of the models for research. All of them did mess up in some way or another, linking to the other Yash's github, having the wrong linkedIn, or having experience that wasn't associated to me. So I wasn't able to make strong conclusions from this small experiment.

---

# Do I care?

I don't know if this was the best benchmark ever, or even a good one to be honest. It just came to mind and I spent an hour making it and writing a blog post. What it did do is show me something new.

This is super cool to me. For the first time, I don't think I could tell that this website was vibecoded with very minimal effort from the first look. Because of that, I think we're going to see a big jump in the way that early stage startups look, and the way that small scale apps are built.

IDK tbh, I'm just speculating. Would love to see what the world does with it..
